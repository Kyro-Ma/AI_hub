## 4. ICLR2026 Oral | 当情感识别不再是分类题：EmotionThinker 让 SpeechLLM 学会“解释情绪”
研究人员提出了一种新的方法叫EmotionThinker，让说话的机器模型不仅能识别人的情绪，还能解释自己是怎么判断的。过去这些模型只能简单地判断情绪是开心还是难过，但新方法让它们像人一样思考，会说出理由。比如当听到一段话时，模型会分析语气、语速和词语选择，然后说明为什么认为这是愤怒或悲伤。这种技术让机器更像人，能理解情绪背后的原因。它特别适合用于客服、教育和心理健康领域，帮助人们更好地理解他人感受。这项研究在国际顶级会议ICLR2026上被选为口头报告，说明其重要性和创新性。这项技术不依赖复杂的算法，而是通过训练让模型学会思考过程，从而提升判断的准确性和可信度。未来它可能被广泛应用于日常生活中，让机器更懂人情世故。

## 9. Context Mode：用 SQLite FTS5 +BM25 将 315 KB MCP 输出降至 5.4 KB（无 LLM 调用）
一个新工具能将原本315KB的代码输出压缩到只有5点4KB，而且不需要调用大模型来完成这个任务。它先把长文本分成小块，用一种叫SQLite的数据库技术建立索引，然后通过一种叫BM25的算法找出和用户问题最相关的部分，只把这些相关片段传给模型。原始内容不会被删除，而是保存在数据库里，如果第一次没找到答案，系统会尝试用不同的方式继续搜索。它还提供了多种回退方法，比如按关键词、按章节或全局搜索，确保不会遗漏重要信息。这个方法不依赖任何大模型，运行速度快，结果稳定可预测。它能和一些常见的编程工具对接，用户可以轻松使用。整个过程只用算法，没有额外的计算开销，节省了大量空间。

## 12. Cloudflare 用 AI 在一周内重写 Next.js：基于 Vite、依赖 Next 测试套件
Cloudflare 公布了一个实验项目 vinext，用人工智能在一周内把 Next.js 的功能搬到 Vite 上，整个过程只用了大约一千一百美元的算力费用。他们利用了 Next.js 原来的大量测试来确保新版本的行为和原版一致，这样 AI 生成的代码就能通过这些测试验证。许多开发者认为这种方法让项目运行更快，编译和热更新也更流畅。但也有不少人担心，这种快速重写可能遗漏一些复杂情况，比如某些简单页面无法启动，或者存在安全漏洞。有人担心未来开源项目会把测试和规则私有化，防止被别人轻易复制。同时社区对 Cloudflare 的做法有不同看法，有人觉得这是打破大公司垄断的好办法，也有人认为语气太自大，忽略了长期维护者的辛苦。这个实验展示了 AI 在软件开发中的潜力，但也带来了关于安全、版权和开源生态的新问题。

## 13. 大语言模型真的会「推理」吗？一项系统性研究梳理 LLM 的结构性推理失败
大语言模型虽然能回答很多问题，但它们在做推理时常常出错。研究人员通过大量实验发现，这些模型在面对需要一步步思考的问题时，容易犯错。比如它们可能把事实记混，或者跳过关键步骤。有些问题看起来简单，模型却给出错误答案。研究还发现，模型的错误往往不是因为知识不够，而是因为内部思考过程不完整或不准确。这种现象被称为结构性推理失败。实验表明，即使模型训练数据非常丰富，它仍然无法像人类一样清晰地分析问题并一步步得出正确结论。目前还没有一种方法能让模型真正学会像人一样思考。OPENAI和其他公司开发的模型在这一方面也存在类似问题。未来需要更多研究来改进模型的思考能力。

## 15. GitHub Copilot SDK使开发人员可以将Copilot CLI的引擎集成到应用中
